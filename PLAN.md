# MathSculpt — план

**Веб-приложение для 3D-печати: поверхности из математических формул.**
Аналог [formula-synth](../formula-synth/) (синтезатор звука из формул), но вместо
звуковой волны — параметрическая поверхность, которую можно скачать как STL и
напечатать. Затравка по 3D-части — `../neural-things/mandelbulb` (Three.js,
экспорт STL, Web Worker с прогрессом); инфраструктура (сборка, тесты, TDD) — по
образцу formula-synth.

## Ключевые решения (согласованы)

- **Ядро — только параметрика** (без SDF/marching cubes): базовая фигура — сетка
  `(u,v) → ℝ³`, поверх неё — стек формул-смещений. Быстро, чистая топология,
  идеально для ваз и тел вращения. Фракталы/TPMS-решётки — сознательно за бортом
  (см. «Не делаем»).
- **LFO — только пространственные**: модуляторы — функции координат (высота `z`,
  азимут `θ`, радиус, их комбинации), не времени. Никакой анимации: превью
  статичен, результат детерминирован, экспорт = то, что на экране. Вместо
  времени — слайдер фазы.
- **Визуализация — Three.js** (npm-зависимость; в отличие от formula-synth здесь
  runtime-зависимость оправдана). Геометрическое ядро от Three.js **не зависит**
  — как DSP отделён от Web Audio в formula-synth.

## Аналогия с formula-synth

| Formula Synth (звук) | MathSculpt (поверхность) |
|---|---|
| Генератор (формула → волна) | Базовая фигура (тело вращения, сфера, тор…) |
| Сумма генераторов в MixBus | Сумма формул-смещений по нормали |
| Эффекты (фильтр, хорус…) | Деформеры (twist, taper, симметрия, сглаживание…) |
| LFO(t) → параметр | LFO(z / θ / z+k·θ) → параметр |
| Матрица модуляции (маршруты, depth, exp) | Та же матрица, но по координатам |
| Осциллограф/спектрограмма | 3D-превью + 2D-график профиля r(z) |
| Запись WAV | Экспорт STL (binary) |
| Пресеты + share-ссылки #s=… | То же (base64url, localStorage) |

## Стек и инфраструктура

TypeScript + Vite + vitest + eslint (запрет `as`-кастов, как в formula-synth) +
playwright для смоука и скриншотов. Сборка в `./docs` (GitHub Pages).

```bash
npm run dev / build / test / lint
npm run check     # tsc + eslint + vitest одной командой
npm run smoke     # headless-Chromium: каждый пресет строится без ошибок консоли,
                  # меш валиден (см. «Тестирование»)
npm run shot      # скриншоты превью в ./shots
```

Процесс — TDD: сначала тест, потом код. UI-тексты — английские (публичное
приложение), документация — как удобно.

## Карта модулей (целевая)

```
src/geo/surface.ts     ЯДРО: построение сетки (u,v) → позиции/нормали/индексы;
                       чистые типы Float32Array/Uint32Array, без Three.js
src/geo/shapes.ts      базовые фигуры: revolve (профиль r(z) → тело вращения,
                       крышки/дно), sphere, torus, superellipsoid, supershape
src/geo/profiles.ts    профили тел вращения: пресеты (ваза, бутыль, чаша, кубок)
                       + профиль из формулы (Фурье-ряд, сплайн по точкам)
src/geo/displace.ts    формулы-смещения: поле d(u,v,p)·normal, стек с весами
src/geo/deform.ts      деформеры позиций: twist, taper, bulge, симметрия k-fold,
                       ...; применяются после смещений
src/geo/mod.ts         пространственные LFO + матрица модуляции (порт идей
                       src/dsp/mod.ts из formula-synth: формы sine/tri/saw/
                       square/S&H, depth, фаза, clamp/exp)
src/geo/normals.ts     пересчёт нормалей после смещений (по сетке, шов u=0/2π)
src/geo/validate.ts    чистые проверки меша: watertight, конечность, bbox,
                       ориентация треугольников, вырожденные треугольники
src/geo/stl.ts         encodeSTL → ArrayBuffer (binary STL, мм)
src/worker/build.ts    Web Worker: тяжёлая перестройка сетки с прогрессом
                       (как mandelbulb-worker), превью низкого разрешения — в
                       главном потоке мгновенно
src/render/scene.ts    Three.js: сцена, свет, orbit-камера, материал; принимает
                       готовые буферы из ядра
src/state/…            схема состояния v1, sanitize без as-кастов, share-ссылки
                       base64url, пресеты пользователя в localStorage
src/formulas.ts        UI-схема: карточки, слайдеры (min/max/step/default)
src/presets.ts         встроенные пресеты (полное состояние)
src/ui/…               карточки, +/- с автоповтором, панель матрицы модуляции,
                       график профиля r(z) (Canvas 2D)
src/main.ts            сборка UI
```

## Конвейер геометрии

```
профиль r(z) или (u,v)-фигура
   → базовые позиции P(u,v) + нормали
   → P' = P + n̂ · Σᵢ wᵢ·dᵢ(u,v)          (стек формул-смещений)
   → деформеры (twist по z, taper, …)     (позиции целиком)
   → пересчёт нормалей, швы, крышки
   → валидация (watertight, NaN, bbox)
   → превью (Three.js) + экспорт STL (мм)
```

Все параметры формул/деформеров могут быть промодулированы пространственным LFO
до вычисления точки — как effectiveParam в formula-synth, только аргумент LFO —
не `t`, а выбранная координата точки.

## Библиотека формул

### Базовые фигуры (носители)

| # | Фигура | Формула | Параметры |
|---|--------|---------|-----------|
| 1 | **Тело вращения** (revolve) | профиль `r(z)`, вращение по θ | профиль (пресет/формула), высота, крышки: дно/верх/оба/нет |
| 2 | **Сфера** | стандартная (θ, φ) | радиус |
| 3 | **Тор** | `(R + r·cos v)·(cos u, sin u), r·sin v` | R, r |
| 4 | **Суперэллипсоид** | сферическое произведение суперэллипсов | e1, e2 (квадрат↔сфера↔октаэдр↔звезда), масштабы |
| 5 | **Supershape 3D** ([суперформула Гилиса](https://en.wikipedia.org/wiki/Superformula)) | `r(φ) = (|cos(mφ/4)/a|^n2 + |sin(mφ/4)/b|^n3)^(−1/n1)`, сферическое произведение двух r₁(θ)·r₂(φ) | m, n1, n2, n3 × 2 набора — цветы, звёзды, ракушки |

Профили тела вращения: пресеты (ваза, амфора, бутыль, чаша, кубок) + профиль
формулой — ряд Фурье `r(z) = r₀ + Σ aₖ·sin(kπz + φₖ)` (3–4 гармоники дают
богатое семейство «кувшинов») + свободный сплайн по контрольным точкам (фаза 3).

### Формулы-смещения (аналог генераторов; складываются с весами)

| # | Смещение | Формула d(u,v) | Даёт |
|---|----------|----------------|------|
| 1 | **Ripples** | `A·sin(a·u + b·v + φ)` | кольца/грани/спирали (a, b целые → замкнуто) |
| 2 | **Lissajous-интерференция** | `A·sin(a·u+φ₁)·sin(b·v+φ₂)` | плетёнка, вафельный узор |
| 3 | **Сферические гармоники** | `A·Yₗₘ(θ, φ)` (веществ.) | органичные лепестки на сфере/вазе |
| 4 | **fBm-шум** (симплекс, детерминир. сид) | `A·Σ noise(2ᵏ·u, 2ᵏ·v)/2ᵏ` | камень, кора, керамика ручной лепки |
| 5 | **Гироид-узор** | `A·sin(sx)·cos(sy)+sin(sy)·cos(sz)+…` в точке поверхности | тканый TPMS-рельеф без решётки ([формула](https://siliconwit.com/education/code-based-mechanical-design/lattice-structures-tpms-additive-manufacturing/)) |
| 6 | **Логистическая карта по z** | `xₙ₊₁=r·xₙ(1−xₙ)` на слой | хаотичные рёбра (мостик к formula-synth) |
| 7 | **S&H-фасеты** | ступени детерминир. хэша по ячейкам (u,v) | низкополигональная огранка |
| 8 | **Bytebeat-рельеф** | целочисленные формулы `((u>>k)&m)·v mod n` | пиксельный орнамент |
| 9 | **Зубцы/пила/меандр** | формы LFO как прямое смещение | шестерни, короны, зиккураты |

### Деформеры (аналог FX-цепочки; применяются к позициям по порядку)

| # | Деформер | Параметры |
|---|----------|-----------|
| 1 | **Twist** | угол/высота (спиральный перекрут вазы) |
| 2 | **Taper** | сужение по z (линейное/степенное) |
| 3 | **Bulge/Pinch** | раздутие середины гауссианой по z |
| 4 | **Симметрия k-fold** | усреднение по повороту 2π/k — «гончарная» правильность |
| 5 | **Smooth** | лапласово сглаживание сетки, 1–5 итераций |
| 6 | **Quantize** | округление позиций к шагу — вокселизация/лоуполи |

## Пространственные LFO и матрица модуляции

Пул из 3–4 LFO. Каждый: **координата-источник** (z, θ, z+k·θ спираль, радиус от
оси, расстояние от центра), **форма** (sine / triangle / saw / square / S&H),
**частота** (периодов на диапазон координаты), **фаза** (слайдер — замена
времени). Маршруты: `LFO → карточка.параметр → depth (−1…+1)`, exp-режим для
«частотных» параметров — всё как в `src/dsp/mod.ts` formula-synth, включая
бейдж ∿ на модулированных слайдерах.

Примеры паттернов: LFO(z)→амплитуда ripples = рёбра, сходящие на нет к горлышку;
LFO(θ, square)→радиус профиля = гранёный стакан; LFO(z+6θ)→twist = витая
колонна; S&H(z)→фаза узора = «сломанная» керамика.

## Экспорт и печатность

- **Binary STL**, единицы — миллиметры; поля «высота, мм» и «диаметр, мм»
  масштабируют модель при экспорте. Имя файла из имени пресета.
- Меш **закрытый и ориентированный** (watertight): у тела вращения — крышки
  веером, шов u=0/2π сшит индексами, а не дублированными вершинами.
- Режимы: «solid» (закрыто всё — для vase-mode печати слайсер сам сделает
  спираль) и «открытый верх + дно» (настоящая полая ваза; толщину стенки даёт
  слайсер). Плоское дно — опция «срезать дно плоскостью» для адгезии.
- Валидатор предупреждает (не блокирует): самопересечения при экстремальных
  смещениях, свесы > 60° (плохо для FDM), нулевая толщина.
- 3MF/OBJ — бэклог.

## UI

Как formula-synth: топбар (пресеты, Save, Share, Export STL, панели-переключатели
Mod/Deform), карточки формул с чекбоксом и слайдерами (+/- с автоповтором),
свёрнуты по умолчанию. Слева/сверху — 3D-превью (orbit-контролы) и график
профиля r(z). Превью перестраивается на лету в низком разрешении (сетка
~96×96 — в главном потоке это единицы миллисекунд), по отпусканию слайдера — в
полном (до ~512×1024) в Web Worker с прогресс-баром (паттерн mandelbulb-worker).

## Тестирование (TDD, по образцу formula-synth)

- **Golden-тесты геометрии**: детерминированные сиды, эталонные вершины
  (подвыборка + хэш буфера) в `tests/golden/*.json`, `UPDATE_GOLDEN=1` для
  осознанной перегенерации. Геометрия = «текущее состояние кода».
- **Свойства** (аналог audio-sanity): для каждой фигуры/смещения/пресета — нет
  NaN/Inf, меш watertight (каждое ребро ровно в 2 треугольниках), нормали
  наружу (положительный объём), bbox в разумных пределах, вырожденных
  треугольников нет.
- **Чистые юниты**: LFO-формы, матрица модуляции, encodeSTL (разобрать буфер
  обратно и сверить), сериализация состояния, профили.
- **Smoke** (playwright): каждый пресет строится и рендерится без ошибок
  консоли, экспорт STL отдаёт валидный буфер; скриншоты в ./shots.
- Three.js-слой в vitest не грузится (WebGL) — как engine в formula-synth:
  тонким держать, проверять смоуком.

## Фазы

   пресет-профилями, ripples-смещение, Three.js-превью, binary STL. Уже можно

2. **Синтезатор**: стек смещений (2, 3, 4, 9), деформеры twist/taper/симметрия,

3. **Модуляция**: пространственные LFO + матрица, бейджи ∿, worker для полного
   разрешения, график профиля, Фурье-профиль.
4. **Полировка**: остальные фигуры (supershape, суперэллипсоид) и смещения
   (гармоники, гироид-узор, bytebeat), smooth/quantize, smoke+скриншоты,
   мобильная вёрстка, деплой на GitHub Pages.

## Не делаем (сознательно)

- **SDF + marching cubes** (мандельбульб, метаболы, TPMS-решётки как объём) —
  решено остаться в параметрике; если захочется, это отдельный второй пайплайн,
  затравка живёт в `../neural-things/mandelbulb`.
- **Анимация/время** — все LFO пространственные, превью статичен.
- Булевы операции над мешами, толщина стенки в приложении (отдаём слайсеру),
  reaction-diffusion (дорого), G-code.

## Открытые вопросы (решим по ходу)

- Редактор профиля точками (drag контрольных точек на графике r(z)) — бэклог.
- Цвет/материал превью — однотонный.
- Лимиты сетки на мобильных (по образцу maxGridNForMemory из mandelbulb).
